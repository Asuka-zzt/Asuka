# 前端设计 — Stage 式界面（参考 AIRI，对齐后端现有能力）

> 目标：搭一套 Vue 3 前端，主视图采用 AIRI 式「左舞台 / 右聊天」布局，
> **为 Live2D 人物预留独立舞台空位**（占位组件，后续挂 Pixi 画布）；
> 前端只暴露**后端当前真实提供的能力**，不为尚不存在的接口造占位页。
>
> 参考 `ref/normal_agent/airi`：
> - 布局：`apps/stage-web/src/pages/index.vue`（背景 → Header → 左 `WidgetStage` / 右 `InteractiveArea`）
> - Live2D：`packages/stage-ui-live2d/src/components/scenes/live2d/Canvas.vue`（透明 Pixi 画布挂容器）
> - 聊天：`packages/stage-layouts/.../InteractiveArea.vue`（ChatHistory + ChatArea + Pinia stores）

---

## 0. 后端能力清单（前端对齐目标）

| 后端接口 | 形态 | 前端对应 |
|---|---|---|
| `GET /health` → `{status, version}` | REST | Header 连接状态徽标（在线点 + 版本号） |
| `WS /ws/{conversation_id}` 流式 | WebSocket | **Chat 视图**：流式逐字对话 |
| `POST /chat {conversation_id, message}` → `{reply}` | REST 非流式 | **Agent 测试台**：表单发一条、看完整回复 |
| `generate_wiki` 工具（Agent 内置） | 经对话触发 | 在 Chat 里自然语言触发（"给 X 生成 wiki"）|
| `POST /generate_wiki`（新增） | REST | **Codebase2Wiki 视图**：表单提交 repo/dir + patterns，独立触发 wiki 生成 |
| 默认 Agent（`id/name/soul/model_id` 硬编码，**无 CRUD 接口**） | 启动时固定 | Agent 测试台**只读展示**默认人格 |

> 没有 Agent CRUD / Prompt 快照 / 知识库接口 → 本期**不做**这些视图（仅在 §12 登记为后续）。

## 1. 范围

**做：**
- Vue 3 + TS + Vite + pnpm + Pinia + vue-router + **UnoCSS**，独立 `frontend/` 目录。
- AIRI 式 `StageLayout`：全屏背景 → Header（含 `/health` 徽标）→ 左**舞台**（Live2D 空位）/ 右**聊天面板**，移动端竖排。
- **Chat 视图（`/`）**：连 `WS /ws/{conversation_id}`，流式 token 渲染、多轮显示；可自然语言触发 `generate_wiki`。
- **Agent 测试台（`/agent`）**：只读展示默认 Agent（name/soul/model_id），下方表单接 **`POST /chat`** 非流式发送、显示 `reply`。
- **Live2DStage 空位组件**：填满左舞台，MVP 显示占位，留好未来挂 `pixi-live2d-display` 的接缝。
- **Codebase2Wiki 视图（`/wiki`）**：独立表单页，参考 `ref/Tutorial-Codebase-Knowledge/main.py`：输入 GitHub repo 或本地目录、项目名、include/exclude patterns、output 目录、语言；提交调用 `POST /generate_wiki`，显示生成结果路径与预览。

**不做（登记 TODO）：**
- Live2D 真实渲染（仅留空位与 composable 接口）。
- Prompt 可视化 / 知识库视图（后端无对应接口）。
- Agent 配置**持久化**（后端无 CRUD，表单字段只读展示，不可保存）。
- 多 Agent 并发气泡（后端当前单 Agent；`Message.agentId` 预留字段不渲染多栏）。
- WS 断线指数退避、Electron 打包、鉴权多用户。

## 5. Live2D 空位设计（关键）

`Live2DStage.vue` —— **预留并填满左舞台，本期占位，未来挂 Pixi 画布**：

```vue
<script setup lang="ts">
import { onMounted, ref } from 'vue'
import StagePlaceholder from './StagePlaceholder.vue'
import { useLive2D } from '@/composables/useLive2D'

defineProps<{ modelUrl?: string }>()
const stageRef = ref<HTMLDivElement>()
const { state } = useLive2D(stageRef)   // 本期 state 恒为 'placeholder'

onMounted(() => {
  // 本期：不挂载，显示占位。
  // 未来：modelUrl 存在时 mount(modelUrl) —— 内部创建透明 Pixi Application
  //       (backgroundAlpha:0)，把 canvas 以 width/height:100% objectFit:contain 填入 stageRef，
  //       参考 AIRI live2d/Canvas.vue。
})
</script>

<template>
  <!-- 这个容器就是 Live2D 的「空位」：透明、填满左舞台、未来 canvas 挂这里 -->
  <div ref="stageRef" relative h-full w-full flex items-center justify-center of-hidden>
    <StagePlaceholder v-if="state === 'placeholder'" />
  </div>
</template>
```

设计要点：透明容器（与 `backgroundAlpha:0` 同思路），尺寸由布局撑开，Pixi 画布 `objectFit:contain` 自适应；

## 7. 状态管理（Pinia）

`stores/chat.ts`（核心）：

```ts
interface Message {
  id: string
  role: 'user' | 'assistant'
  agentId?: string          // 预留多人格；当前恒为 'default'
  content: string
  streaming?: boolean
}
state:   { conversationId, messages: Message[], sending, connected }
actions: send(text) / appendToken(token) / finalize() / setError(msg)
```

- `health.ts`：轮询 `GET /health`，存 `online` / `version`。
- `ui.ts`：主题 / 当前视图。

## 8. 与后端对接

### 8.1 WebSocket（`routes/ws.py`）

```
连接：ws://<host>/ws/{conversationId}
发送：{ "message": "用户输入" }
接收：{ "type":"token", "content":"增量" } | { "type":"done" } | { "type":"error", "content":"…" }
```

`useChatSocket.ts`：建连、`send(text)`、token→`chat.appendToken`、done→`finalize`、error→`setError`、基础重连。

### 8.2 REST（`routes/chat.py`、`main.py`）

```
POST /chat { conversation_id, message } → { reply }     # Agent 测试台
GET  /health → { status, version }                       # Header 徽标轮询
```

`api/client.ts` 封装两者。

### 8.3 Vite dev proxy（避免 CORS）

```ts
server: {
  proxy: {
    '/ws':     { target: 'ws://127.0.0.1:8000', ws: true },
    '/chat':   { target: 'http://127.0.0.1:8000' },
    '/health': { target: 'http://127.0.0.1:8000' },
  },
}
```

## 9. 数据流（一次流式对话）

```
ChatInput 输入 → chat.send(text)
  → push {role:user} + push {role:assistant, streaming:true}
  → useChatSocket.send({message:text})
  → WS ← {type:token} ×N → chat.appendToken → 气泡逐字增长
  → WS ← {type:done}     → chat.finalize    → 气泡定稿
  （触发 wiki 时：工具执行期间无 token，结束后摘要整段流回；多轮由后端 checkpointer 续上下文）
```

## 7. Codebase2Wiki 视图设计（`/wiki`）

参考 `ref/Tutorial-Codebase-Knowledge/main.py` 的 argparse 参数与流程，设计独立表单页（不使用 StageLayout）：

**表单字段**（严格对齐参考 CLI）：
- 源类型（radio/tab）：GitHub Repo / Local Dir（互斥，必选其一）
  - GitHub：输入框 `--repo` URL + 可选 `--token`
  - Local：输入框 `--dir` 路径（支持拖拽或文件选择器）
- 项目名 `--name`（text，可选；留空则自动从路径/URL 推导）
- Include patterns `--include`（多行 textarea 或 tag input；默认 DEFAULT_INCLUDE_PATTERNS）
- Exclude patterns `--exclude`（多行 textarea 或 tag input；默认 DEFAULT_EXCLUDE_PATTERNS）
- 最大文件大小 `--max-size`（number，默认 100000）
- 输出目录 `--output`（text，默认 "output"）
- 语言 `--language`（select：english / 中文，默认 english）
- 最大抽象数 `--max-abstractions`（number，默认 10）
- 禁用缓存 `--no-cache`（checkbox）

**UI 布局**（参考典型 codebase-to-doc 工具）：
- 顶部标题 + 说明
- 左侧/上方：源选择卡片（GitHub / 本地）
- 中部：参数表单（grid 两列，响应式）
- 底部：提交按钮（loading 态）
- 结果区（提交后显示）：生成状态、output_dir 路径、生成文件列表、打开/下载按钮

**交互流程**：
1. 填写表单 → 提交
2. 调用 `POST /generate_wiki`（body 匹配 shared dict 字段）
3. 显示 loading + 进度提示（后端若支持流式进度则用 WS）
4. 成功：展示 `final_output_dir` + 文件树预览；失败 toast + 错误信息

**新增文件**：
- `views/CodebaseWikiView.vue`（表单 + 结果展示）
- `router/index.ts`：添加 `/wiki` 路由（导航中置于 Chat/Agent 之后）
- `AppHeader.vue`：新增 “Wiki 生成” 链接
- `api/client.ts`：`generateWiki(payload: GenerateWikiRequest): Promise<GenerateWikiResponse>`
- `types/wiki.ts`：定义请求/响应类型（对齐 shared 字典）

**后端依赖**：需实现 `POST /generate_wiki`（包装 LangGraph 子图或直接调用工具），返回 `{status, output_dir, files, chapter_order}` 等。

**验收**：表单提交后能触发 wiki 生成，后端返回结果路径，前端正确展示。
