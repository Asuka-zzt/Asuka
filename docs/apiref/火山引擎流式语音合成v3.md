# 火山引擎流式语音合成 V3 接入参考

> 来源：火山引擎「语音合成大模型 API」V3 WebSocket 双向流式文档。本文为本项目接入用清洗版，去除了网页导出残留、MDX 包装和冗长示例，保留协议与实现相关信息。

## API 选择

| 接口 | 协议 | 推荐场景 | 文档 |
|---|---|---|---|
| `wss://openspeech.bytedance.com/api/v3/tts/bidirection` | WebSocket 双向流式 | 实时交互，支持文本流式输入、音频流式输出 | [V3 WebSocket 双向流式](https://www.volcengine.com/docs/6561/1329505) |
| `wss://openspeech.bytedance.com/api/v3/tts/unidirectional/stream` | WebSocket 单向流式 | 一次性输入文本，流式输出音频 | [V3 WebSocket 单向流式](https://www.volcengine.com/docs/6561/1719100) |
| `https://openspeech.bytedance.com/api/v3/tts/unidirectional` | HTTP Chunked | 一次性输入文本，HTTP chunked 输出音频 | [V3 HTTP Chunked 单向流式](https://www.volcengine.com/docs/6561/1598757?lang=zh#_2-http-chunked%E6%A0%BC%E5%BC%8F%E6%8E%A5%E5%8F%A3%E8%AF%B4%E6%98%8E) |
| `https://openspeech.bytedance.com/api/v3/tts/unidirectional/sse` | HTTP SSE | 一次性输入文本，SSE 输出音频 | [V3 SSE 单向流式](https://www.volcengine.com/docs/6561/1598757?lang=zh#_3-sse%E6%A0%BC%E5%BC%8F%E6%8E%A5%E5%8F%A3%E8%AF%B4%E6%98%8E) |

本项目实时语音优先使用 `bidirection`：LLM token 可以边生成边发给 TTS，服务端同时流式返回音频，首响更低。

## 调用流程

双向流式接口支持同一个 WebSocket 连接下串行创建多个 session，但不支持同一连接内并发多个 session。

推荐流程：

1. 建立 WebSocket：携带鉴权 headers 连接 `wss://openspeech.bytedance.com/api/v3/tts/bidirection`。
2. 发送 `StartConnection`。
3. 收到 `ConnectionStarted` 后，发送 `StartSession`，其中携带音色、音频格式、采样率等 TTS 配置。
4. 持续发送 `TaskRequest`，其中携带当前要合成的文本片段。
5. 没有更多文本时，立即发送 `FinishSession`。
6. 等待 `SessionFinished`。如果还要继续合成，重新发送 `StartSession` 开启下一轮 session。
7. 不再合成时，发送 `FinishConnection` 断开连接。

注意事项：

- TTS 服务参数只在 `StartSession` 生效。
- 每次发送的文本只在对应 `TaskRequest` 生效。
- 同一连接复用时，必须等上一轮 `SessionFinished` 后再创建下一轮 session。
- `CancelSession` 可用于客户端主动放弃当前 session，推荐在收到 `SessionStarted` 后、发送 `FinishSession` 前使用。收到 `SessionCanceled` 后如需继续合成，需要重新 `StartSession`。

## 建连与鉴权

### 新版控制台

推荐使用新版控制台的 API Key 鉴权。

| Header | 必须 | 说明 | 示例 |
|---|---:|---|---|
| `X-Api-Key` | 是 | 火山引擎控制台获取的 API Key | `your-api-key` |
| `X-Api-Resource-Id` | 是 | 资源信息 ID，用于选择模型版本和计费商品 | `seed-tts-2.0` |
| `X-Api-Connect-Id` | 否 | 连接追踪 ID，建议传 UUID，便于排查问题 | `67ee89ba-7050-4c04-a3d7-ac61a63499b3` |

```python
headers = {
    "X-Api-Key": "your-api-key",
    "X-Api-Resource-Id": "seed-tts-2.0",
}
```

### 旧版控制台

旧版控制台使用 App ID 与 Access Key 鉴权，建议尽快迁移到新版 API Key。

| Header | 必须 | 说明 | 示例 |
|---|---:|---|---|
| `X-Api-App-Id` | 是 | 火山引擎控制台获取的 APP ID | `123456789` |
| `X-Api-Access-Key` | 是 | 火山引擎控制台获取的 Access Token | `your-access-key` |
| `X-Api-Resource-Id` | 是 | 资源信息 ID，用于选择模型版本和计费商品 | `seed-tts-2.0` |
| `X-Api-Connect-Id` | 否 | 连接追踪 ID，建议传 UUID，便于排查问题 | `67ee89ba-7050-4c04-a3d7-ac61a63499b3` |

```python
headers = {
    "X-Api-App-Id": "123456789",
    "X-Api-Access-Key": "your-access-key",
    "X-Api-Resource-Id": "seed-tts-2.0",
}
```

### Resource ID

| Resource ID | 模型/商品 |
|---|---|
| `seed-tts-2.0` | 豆包语音合成模型 2.0，仅支持 TTS 2.0 音色；计费商品为语音合成 2.0 字符版 |
| `seed-tts-1.0` | 豆包语音合成模型 1.0，仅支持 TTS 1.0 音色；计费商品为语音合成 1.0 字符版 |
| `seed-tts-1.0-concurr` | 豆包语音合成模型 1.0 并发版；计费商品为语音合成 1.0 并发版 |
| `seed-icl-2.0` | 豆包声音复刻 2.0；计费商品为声音复刻 2.0 字符版 |
| `seed-icl-1.0` | 豆包声音复刻 1.0；计费商品为声音复刻 1.0 字符版 |
| `seed-icl-1.0-concurr` | 豆包声音复刻 1.0 并发版；计费商品为声音复刻 1.0 并发版 |

### 额外 Header

| Header | 必须 | 说明 |
|---|---:|---|
| `X-Control-Require-Usage-Tokens-Return` | 否 | 控制是否在 `SessionFinished` 事件中返回用量数据。可传 `*` 返回已支持用量，也可传 `text_words`；多个标记用逗号分隔 |

WebSocket 握手成功后，响应 header 中会返回 `X-Tt-Logid`，建议记录用于排查问题。

## WebSocket 二进制协议

WebSocket 使用二进制帧传输。整数均使用大端序。

一帧由以下部分组成：

- 至少 4 字节 header：协议版本、header size、message type、序列化方式、压缩方式。
- 可选字段：event、connection id、session id、error code 等。
- payload size：payload 字节长度。
- payload：JSON、音频数据或错误对象。

### 固定 Header

| Byte | 高 4 bit | 低 4 bit | 说明 |
|---|---|---|---|
| `0` | Protocol version | Header size | 当前固定为 `0b0001_0001`，即 v1、4 字节 header |
| `1` | Message type | Message type specific flags | 是否为 client/server、是否携带 event 等 |
| `2` | Serialization method | Compression method | JSON/raw 与是否 gzip |
| `3` | Reserved | Reserved | 固定保留 |

### Message Type

| 值 | 含义 | Flags | 是否含 Event | 备注 |
|---|---|---|---:|---|
| `0b0001` | Full-client request | `0b0100` | 是 | 客户端完整请求，JSON payload |
| `0b1001` | Full-server response | `0b0100` | 是 | 服务端完整响应，JSON 或混合数据 |
| `0b1011` | Audio-only response | `0b0100` | 是 | 服务端音频响应 |
| `0b1111` | Error information | none | 否 | 错误帧 |

### Serialization Method

| 值 | 含义 |
|---|---|
| `0b0000` | Raw，主要用于二进制音频 |
| `0b0001` | JSON，主要用于文本消息 |

### Compression Method

| 值 | 含义 |
|---|---|
| `0b0000` | 无压缩 |
| `0b0001` | gzip |

## Event 定义

| Code | 名称 | 类型 | 方向 | 说明 |
|---:|---|---|---|---|
| `1` | `StartConnection` | Connect | 上行 | WebSocket upgrade 后声明创建连接 |
| `2` | `FinishConnection` | Connect | 上行 | 结束连接 |
| `50` | `ConnectionStarted` | Connect | 下行 | 成功建连 |
| `51` | `ConnectionFailed` | Connect | 下行 | 建连失败 |
| `52` | `ConnectionFinished` | Connect | 下行 | 连接结束成功 |
| `100` | `StartSession` | Session | 上行 | 创建会话 |
| `101` | `CancelSession` | Session | 上行 | 取消会话 |
| `102` | `FinishSession` | Session | 上行 | 声明结束会话 |
| `150` | `SessionStarted` | Session | 下行 | 成功开始会话 |
| `151` | `SessionCanceled` | Session | 下行 | 会话已取消 |
| `152` | `SessionFinished` | Session | 下行 | 会话已结束 |
| `153` | `SessionFailed` | Session | 下行 | 会话失败 |
| `200` | `TaskRequest` | Data | 上行 | 传输待合成文本 |
| `350` | `TTSSentenceStart` | Data | 下行 | 句子开始 |
| `351` | `TTSSentenceEnd` | Data | 下行 | 句子结束 |
| `352` | `TTSResponse` | Data | 下行 | 返回音频内容 |

## 请求 Payload

`StartSession` 传 TTS 服务配置，`TaskRequest` 传本次待合成文本。

### 顶层字段

| 字段 | 必须 | 类型 | 说明 | 默认值 |
|---|---:|---|---|---|
| `user.uid` | 否 | string | 用户 ID | 空 |
| `event` | 是 | number | 请求事件码 | 空 |
| `namespace` | 否 | string | 请求方法 | `BidirectionalTTS` |
| `req_params.text` | `TaskRequest` 必须 | string | 输入文本，双向流式不支持 SSML | 空 |
| `req_params.model` | 否 | string | 声音复刻 2.0 可用：`seed-tts-2.0-standard` 或 `seed-tts-2.0-expressive` | `seed-tts-2.0-standard` |
| `req_params.speaker` | 是 | string | 发音人 ID | 空 |
| `req_params.audio_params` | 是 | object | 音频参数 | 空 |
| `req_params.additions` | 否 | JSON string/object | 扩展参数 | 空 |
| `req_params.mix_speaker` | 否 | object | 混音参数，仅 TTS 1.0 音色适用 | 空 |

### `audio_params`

| 字段 | 类型 | 说明 | 默认值 |
|---|---|---|---|
| `format` | string | 音频编码：`mp3`、`ogg_opus`、`pcm`。流式场景不建议传 `wav`，否则可能多次返回 wav header | `mp3` |
| `sample_rate` | number | 采样率，可选 `8000`、`16000`、`22050`、`24000`、`32000`、`44100`、`48000` | `24000` |
| `bit_rate` | number | 比特率。MP3/OGG 建议主动设置，否则默认值可能导致音质损耗 | 空 |
| `emotion` | string | 情感，例如 `angry`。仅部分音色支持 | 空 |
| `emotion_scale` | number | 情绪值，范围 `1~5`，非线性增长 | `4` |
| `speech_rate` | number | 语速，范围 `[-50, 100]`，`100` 表示 2 倍速，`-50` 表示 0.5 倍速 | `0` |
| `loudness_rate` | number | 音量，范围 `[-50, 100]`，`100` 表示 2 倍音量，`-50` 表示 0.5 倍音量。mix 音色暂不支持 | `0` |
| `enable_timestamp` | bool | 返回句级字/词时间戳，仅 TTS 1.0、ICL 1.0 生效 | `false` |
| `enable_subtitle` | bool | 返回字幕事件 `TTSSubtitle`，仅 TTS 2.0、ICL 2.0 生效 | `false` |

### `additions`

| 字段 | 类型 | 说明 | 默认值 |
|---|---|---|---|
| `silence_duration` | number | 句尾增加静音时长，范围 `0~30000ms` | `0` |
| `enable_language_detector` | bool | 自动识别语种 | `false` |
| `disable_markdown_filter` | bool | `true` 时解析并过滤 Markdown；`false` 时不解析不过滤，可能读出符号 | `false` |
| `disable_emoji_filter` | bool | `true` 时不过滤 emoji | `false` |
| `mute_cut_threshold` | string | 静音判断阈值，需配合 `mute_cut_remain_ms` | 空 |
| `mute_cut_remain_ms` | string | 静音裁剪后保留时长。MP3 句首仍可能保留 100ms 内静音；TTS 2.0/ICL 2.0 暂不支持 | 空 |
| `enable_latex_tn` | bool | 是否播报 LaTeX 公式，需 `disable_markdown_filter=true` | `false` |
| `latex_parser` | string | `v2` 时使用 LID 能力解析公式，需 `disable_markdown_filter=true` | 空 |
| `max_length_to_filter_parenthesis` | int | 是否过滤括号内容，`0` 不过滤，`100` 过滤 | `100` |
| `explicit_language` | string | 明确语种，例如 `zh-cn`、`en`、`ja`、`es-mx`、`id`、`pt-br`、`ko` 等，具体范围依音色/模型而定 | 空 |
| `context_language` | string | 给模型提供参考语种，例如 `id`、`es`、`pt` | 空 |
| `explicit_dialect` | string | 明确方言。当前仅部分音色支持 `dongbei`、`shaanxi`、`sichuan` | 空 |
| `unsupported_char_ratio_thresh` | float | 不支持字符比例阈值，超过则返回错误，最大 `1.0` | `0.3` |
| `aigc_watermark` | bool | 是否在合成结尾增加音频节奏标识 | `false` |
| `aigc_metadata` | object | 在音频 header 加入元数据隐式水印，支持 mp3/wav/ogg_opus | 空 |
| `cache_config` | object | 开启相同文本缓存，缓存保留 1 小时；缓存返回数据不附带时间戳 | 空 |
| `post_process.pitch` | int | 音调，范围 `[-12, 12]` | `0` |
| `context_texts` | string list | TTS 2.0/ICL 2.0 表现力增强可用；语音指令，当前仅列表第一个值有效，不计费 | `null` |
| `section_id` | string | TTS 2.0/ICL 2.0 可用；多轮会话 ID，用于复用历史上下文 | 空 |
| `use_tag_parser` | bool | 声音复刻 2.0 表现力增强可用；开启语音标签 COT 解析 | `false` |

常见 `additions` 示例：

```json
{
  "disable_markdown_filter": true,
  "disable_emoji_filter": false,
  "explicit_language": "zh-cn",
  "cache_config": {
    "text_type": 1,
    "use_cache": true,
    "use_segment_cache": true
  }
}
```

### 混音请求示例

```json
{
  "user": {
    "uid": "12345"
  },
  "event": 100,
  "namespace": "BidirectionalTTS",
  "req_params": {
    "speaker": "custom_mix_bigtts",
    "audio_params": {
      "format": "mp3",
      "sample_rate": 24000
    },
    "mix_speaker": {
      "speakers": [
        {
          "source_speaker": "zh_male_bvlazysheep",
          "mix_factor": 0.3
        },
        {
          "source_speaker": "BV120_streaming",
          "mix_factor": 0.3
        },
        {
          "source_speaker": "zh_male_ahu_conversation_wvae_bigtts",
          "mix_factor": 0.4
        }
      ]
    }
  }
}
```

混音限制：

- 最多支持 3 个音色混音。
- `mix_factor` 之和必须为 `1`。
- 使用 mix 时，`req_params.speaker` 需要为 `custom_mix_bigtts`。
- 音色风格差异较大的两个音色以 `0.5/0.5` 混合时可能偶发跳变，建议避免。

## 响应 Payload

### 建连响应

- HTTP 状态码 `200`：建连成功。
- 非 `200`：建连失败，body 中提供错误原因。

### WebSocket 响应

| 类型 | 说明 |
|---|---|
| 文本帧 | 主要反馈异常错误信息 |
| 二进制帧 | 按协议结构化返回正常响应和一般错误信息 |

正常响应 payload 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `data` | bytes | 返回的二进制数据包 |
| `event` | number | 返回事件类型 |
| `res_params.text` | string | 文本分句后的句子 |

错误响应帧：

| 字段 | 说明 |
|---|---|
| Byte 1 | 固定为 `0b1111_0000`，即 error information |
| Byte 2 | JSON + no compression |
| Byte 4-7 | error code |
| Payload | 错误消息对象 |

## 时间戳与字幕

| 能力 | TTS 1.0 / ICL 1.0 | TTS 2.0 / ICL 2.0 |
|---|---|---|
| 事件交互 | 多个子句会多次返回 `TTSSentenceStart` 和 `TTSSentenceEnd`；开启字幕后，字幕跟随 `TTSSentenceEnd` 返回 | 多个子句仅返回一次 `TTSSentenceStart` 和 `TTSSentenceEnd`；开启字幕后会多次返回 `TTSSubtitle` |
| 返回时机 | 一个子句的时间戳返回后才开始返回下一句音频 | 一句音频合成后不一定立即返回字幕；字幕识别不阻塞音频合成，可能字幕返回时下一句音频已开始返回 |
| 打轴文本 | `text` 为原文，`words[].word` 为 TN 后文本 | `text` 为原文，`words[].word` 为原文 |
| 语种 | 中、英；不支持小语种和方言 | 中、英；不支持小语种和方言 |
| LaTeX | `enable_latex_tn=true` 时有字幕返回 | `enable_latex_tn=true` 时无字幕返回，接口不报错 |
| SSML | `req_params.ssml` 不为空时有字幕返回 | `req_params.ssml` 不为空时无字幕返回，接口不报错 |

## 典型帧结构

### `StartConnection`

| 字段 | 值 |
|---|---|
| Header | `0x11 0x14 0x10 0x00` |
| Event | `1` |
| Payload | `{}` |

### `StartSession`

| 字段 | 值 |
|---|---|
| Header | `0x11 0x14 0x10 0x00` |
| Event | `100` |
| Session ID | 客户端生成，不能为空 |
| Payload | `tts_session_meta`，包含 `user`、`req_params` |

### `TaskRequest`

| 字段 | 值 |
|---|---|
| Header | `0x11 0x14 0x10 0x00` |
| Event | `200` |
| Session ID | 当前 session ID |
| Payload | `{ "req_params": { "text": "..." } }` |

### `FinishSession`

| 字段 | 值 |
|---|---|
| Header | `0x11 0x14 0x10 0x00` |
| Event | `102` |
| Session ID | 当前 session ID |
| Payload | `{}` |

### `SessionFinished`

| 字段 | 值 |
|---|---|
| Header | `0x11 0x94 0x10 0x00` |
| Event | `152` |
| Session ID | 当前 session ID |
| Payload | `{ "status_code": 20000000, "message": "ok", "usage": { "text_words": 4 } }` |

`usage` 仅在请求 header 携带 `X-Control-Require-Usage-Tokens-Return` 时返回。

## 实现提示

- 音频格式若用于低延迟播放，可优先评估 `pcm`；若使用 `mp3`/`ogg_opus`，建议显式配置 `bit_rate`。
- 前端聊天文本含 Markdown、emoji、控制标签时，进入 TTS 前应先清洗；服务端也可以开启 `disable_markdown_filter=true` 辅助过滤 Markdown。
- 记录 `X-Tt-Logid`、`X-Api-Connect-Id`、`session_id`，便于定位建连、会话和音频返回问题。
- 本项目 API Key、App ID、Access Key 只应从环境变量读取，不应写入文档示例以外的代码。
