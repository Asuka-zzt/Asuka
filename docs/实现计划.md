# Asuka 实现计划（MVP 路线图）

> 目标：最快搭建一个可对话的 Agent，支持工具调用，用"代码项目 → Wiki 文档生成"功能验收。

## 核心决策

- 用 LangGraph 预构建的 **`create_react_agent`**（一行启动），不手写 StateGraph。
- 先做**单 Agent** 跑通，再扩展到多 Agent 并发调度。
- 短期记忆直接用 LangGraph 内置 **`AsyncSqliteSaver`** checkpointer，`thread_id` = `conversation_id`。
- MVP 阶段跳过：前端、插件系统、中/长期记忆、`session/` 层。

## Phase 2 — 最小可对话 Agent

**目标：** WebSocket 连接后能多轮对话，上下文不丢失。

```
asuka/core/agent/model.py        AgentConfig（name / soul / model_id）
asuka/core/graph/dispatch.py     create_react_agent(llm, tools, checkpointer)
asuka/core/graph/checkpointer.py AsyncSqliteSaver（对话持久化）
asuka/api/provider/__init__.py   get_llm(model_id) → ChatOpenAI / ChatAnthropic
asuka/routes/ws.py               WebSocket → astream_events → 流式推送
asuka/routes/chat.py             POST /chat（非流式，调试用）
```

核心代码模式：

```python
# dispatch.py
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.sqlite.aio import AsyncSqliteSaver

async def build_agent(config: AgentConfig, tools: list):
    checkpointer = AsyncSqliteSaver.from_conn_string("./data/sessions.db")
    return create_react_agent(
        model=get_llm(config.model_id),
        tools=tools,
        prompt=config.soul,
        checkpointer=checkpointer,
    )
```

**完成标志：** WebSocket 发消息，agent 流式回复；第二条消息能记得第一条内容。

## Phase 3 — Wiki 生成工具

**目标：** 告诉 agent"给 /path/to/project 生成 wiki"，自动调用工具输出文档。

```
asuka/core/tools/wiki_generator.py   @tool 实现
asuka/core/tools/registry.py         工具注册表
```

工具内部流程（单工具，不拆子图）：

```
@tool wiki_generator(project_path: str) -> str
  1. walk_dir()          遍历目录，收集源码文件，跳过 node_modules/__pycache__/.git
  2. build_tree()        生成目录树字符串
  3. summarize_modules() 按包/模块分批（每批 ≤ 5 文件），LLM 生成模块摘要
  4. gen_index()         LLM 生成 Wiki 首页（项目概述 + 模块导航）
  5. gen_pages()         每个模块生成一个 Markdown 页
  6. write_output()      写入 {project_path}/wiki/ 目录
  7. return              摘要报告（生成了哪些文件）
```

**注意：** 单次 LLM 调用处理整个项目会超 context，步骤 3 必须**分批处理**。

**完成标志：** `{project_path}/wiki/` 下生成 `index.md` + 各模块 `.md`，内容准确反映代码结构。

## Phase 4 — 验收

用 ref/ 下某个参考项目实际测试：

```
WS → "帮我给 ref/spec_agent/TradingAgents 生成 wiki 文档"
```

**验收标准：**
- ✓ agent 正确识别意图并调用 `wiki_generator` 工具
- ✓ `wiki/` 目录生成，`index.md` 有清晰的模块导航
- ✓ 至少 3 个模块页面内容准确
- ✓ 多轮对话：追问"graph 模块是做什么的"，agent 能结合上下文回答

## 后续扩展（MVP 之后）

| 功能 | 触发时机 |
|------|----------|
| 多 Agent 并发（`StateGraph` + `Send`） | 单 Agent 跑通后 |
| `session/manager.py` 会话运行时管理 | 需要管理多会话生命周期时 |
| 中/长期记忆（摘要 + RAG） | 知识库功能开发时 |
| 插件系统动态加载 | 工具数量增长、需要热插拔时 |
| Vue 前端 + Live2D | 后端 API 稳定后 |
