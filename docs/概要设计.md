# Asuka 概要设计

## 1. 系统概述

Asuka是一款可观测,可编排,自进化,多场景的多AGENT AI聊天应用,支持live2d数字人,插件扩展,多层记忆(本地数据库搭建,基于agentic graphrag),主要场景包括:教育,金融,coding等。

---

## 3. 模块划分

### 3.1 Agent 管理模块

负责 Agent 的全生命周期管理。

**核心实体 Agent**

```
Agent {
  id          : string       // 唯一标识
  name        : string       // 显示名称
  avatar      : string       // 头像 URL 或 Live2D 模型路径
  soul        : string       // 人格 System Prompt（灵魂）
  model       : ModelConfig  // 使用的 LLM 及参数
  skills      : Skill[]      // 绑定的技能列表
  tools       : Tool[]       // 绑定的工具列表
  memoryConfig: MemoryConfig // 记忆策略配置
  createdAt   : timestamp
}

ModelConfig {
  provider    : string  // 'openai' | 'anthropic' | 'ollama' | ...
  modelId     : string
  temperature : number
  maxTokens   : number
}
```

**功能点**
- 创建 / 编辑 / 删除 Agent
- Agent 配置导入导出（JSON）
- Agent 预设模板市场（内置若干初始人格）

---

### 3.2 多 Agent 调度模块

实现"一问多答"的核心调度逻辑。

**流程**

```
用户发送消息
    │
    ▼
调度器获取当前会话激活的 Agent 列表
    │
    ▼
并发向各 Agent 发起 LLM 请求（Promise.all + 流式输出）
    │
    ▼
各 Agent 回复通过 WebSocket 独立推流到前端
    │
    ▼
前端按 Agent 分栏/分气泡展示回复
```

**关键设计**
- 每个 Agent 维护独立的上下文（System Prompt + 记忆 + 历史）
- 支持"点名回复"：用户可在消息中 @AgentName，只触发该 Agent
- 支持"接力模式"：Agent A 的回复作为 Agent B 的输入（可选链式调用）

---

### 3.3 Prompt 可视化模块

让用户完全看清当前一次调用的完整 Prompt 构成及 Token 消耗。

**Prompt 构成层次（按注入顺序）**

```
┌─────────────────────────────────────┐
│ ① Soul（人格 System Prompt）        │
│ ② Skills（技能指令注入）            │
│ ③ Tools（工具定义）                 │
│ ④ Long-term Memory（RAG 检索结果）  │
│ ⑤ Mid-term Memory（会话摘要）       │
│ ⑥ Short-term Memory（近 N 轮对话）  │
│ ⑦ User Message                     │
└─────────────────────────────────────┘
```

**可视化面板**
- 折叠/展开各层内容
- 每层显示该层 Token 数与占总量百分比
- 总 Token 进度条，接近上限时高亮警告
- 可视窗口显示完整的当次发送内容（与模型实际收到的完全一致）

---

### 3.4 多层记忆模块

```
短期记忆（Short-term）
  - 存储：内存（会话内）
  - 内容：当前会话最近 N 轮消息原文
  - 策略：滑动窗口，超出自动滚出

中期记忆（Mid-term）
  - 存储：关系型 DB
  - 内容：每隔 M 轮由 LLM 自动生成的会话摘要
  - 策略：摘要压缩，保留关键信息

长期记忆（Long-term）
  - 存储：向量 DB（Qdrant）+ 关系型 DB 元数据
  - 内容：用户主动添加的知识库文档 + 自动沉淀的重要事实
  - 策略：RAG 检索，相似度阈值过滤后注入 Prompt
```

**知识库管理**
- 支持上传 PDF / TXT / Markdown / 网页 URL
- 文档自动分块 → Embedding → 入向量库
- 支持多知识库，Agent 可绑定指定知识库

---

### 3.5 插件系统模块

**插件类型**

| 类型 | 说明 | 示例 |
|------|------|------|
| Tool Plugin | 为 Agent 提供可调用工具 | 天气查询、代码执行、网页搜索 |
| Skill Plugin | 向 Prompt 注入技能指令 | 角色扮演规则、写作风格 |
| UI Plugin | 扩展前端界面组件 | 自定义消息气泡、侧边面板 |

**插件规范（Manifest）**

```json
{
  "id": "plugin-weather",
  "name": "天气查询",
  "version": "1.0.0",
  "type": "tool",
  "entry": "./index.js",
  "permissions": ["network"],
  "schema": { /* JSON Schema for tool call */ }
}
```

**执行隔离**：Tool Plugin 运行在 Node.js vm 沙箱中，限制文件系统与网络权限。

---

### 3.6 虚拟形象模块（Live2D）

**参考 airi 架构**

```
前端 Live2D 画布（PixiJS + pixi-live2d-display）
    │
    ├── 状态机（idle / thinking / talking / emotion）
    │       由 Agent 回复内容中的情绪标签驱动
    │
    └── 口型同步（Lipsync）
            ├── TTS 音频流 → 振幅分析 → 口型参数
            └── 支持多 TTS 后端（Edge TTS / VITS / OpenAI TTS）
```

**情绪驱动**：LLM 回复末尾附带结构化情绪标注 `[emotion:happy]`，前端解析后驱动对应 Live2D 动作与表情参数。

## 4. 数据模型

```

Plugin          (id, name, type, manifest, enabled, installed_at)
```

## 5. 核心数据流

### 5.1 发送消息完整流程

```
用户输入
  → 调度器获取激活 Agent 列表
  → 各 Agent 并发执行：
      ① 短期记忆（最近N轮）
      ② 中期记忆（会话摘要）
      ③ RAG 检索长期记忆
      ④ 构建完整 Prompt（Soul + Skills + Tools + Memory + History + User）
      ⑤ 记录 PromptSnapshot（供可视化面板读取）
      ⑥ 流式调用 LLM API
      ⑦ 解析情绪标签，推送 Live2D 状态
  → 各 Agent 回复流式推送前端展示
  → 保存 Message + PromptSnapshot 到 DB
  → 异步：判断是否触发中期记忆摘要
```

## 7. 关键风险与对策

| 风险 | 对策 |
|------|------|
| 多 Agent 并发成本高 | 用户可设置激活 Agent 数量上限；支持本地 Ollama 降低费用 |
| 向量 DB 部署门槛高 | 优先内置轻量级 in-memory 向量搜索（HNSWlib），Qdrant 作为可选升级 |
| 插件安全 | vm2/isolated-vm 沙箱 + 显式 Permission 声明 + 用户二次确认 |
| Live2D 版权 | 仅内置 Free-to-use 模型；商业模型由用户自行提供文件 |
| Token 超限 | Prompt 构建时强制预算分配，各层有软/硬上限，超限时截断非核心层 |
