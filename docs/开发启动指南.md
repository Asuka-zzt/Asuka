## 3. 环境变量配置（.env）

复制模板并填写至少一个 LLM Provider Key（否则仅 health 可用，chat 会报错）：

```bash
cp .env.example .env
```

关键变量（来自 `asuka/config.py` + `.env.example`）：

```env
HOST=127.0.0.1
PORT=8000
DEBUG=true                 # 开发建议 true（uvicorn reload）

DEFAULT_MODEL=deepseek-chat
DEEPSEEK_API_KEY=sk-xxx
DEEPSEEK_BASE_URL=https://api.deepseek.com
# OPENAI_API_KEY=...
# ANTHROPIC_API_KEY=...

DATA_DIR=./data
SESSION_DB=./data/sessions.db
WIKI_OUTPUT_DIR=./output
```

- 所有 Key **只从环境变量**读取，严禁硬编码（config 使用 pydantic-settings）。
- 前端额外支持 VITE_ 变量（用于独立部署，非 proxy 场景）：
  - `VITE_API_BASE=http://127.0.0.1:8000`
  - `VITE_WS_BASE=ws://127.0.0.1:8000`

## 4. 后端启动方式

### 4.1 标准开发启动（推荐）

```bash
# 根目录
uv sync                     # 同步 pyproject.toml 依赖（含 dev 组可选）
uv run python -m asuka.main
```

- 等价入口：`uv run uvicorn asuka.main:app --reload`
- 内部实现（`asuka/main.py:42`）：
  - `get_settings()` 读 .env
  - `uvicorn.run("asuka.main:app", host, port, reload=debug)`
- 启动日志关键：`Application startup complete.`
- 生命周期：lifespan 中 yield 后 `close_checkpointer()`（释放 SQLite 连接）。

### 4.2 验证后端

```bash
curl http://127.0.0.1:8000/health
# {"status":"ok","version":"0.1.0"}
```

其他接口（见设计）：

- `WS /ws/{conversation_id}` — 流式主入口
- `POST /chat` — 非流式测试（AgentTestView 用）
- `POST /generate_wiki` — 独立 wiki 生成
- `POST /api/tts` / `/api/tts/stream`

## 5. 前端启动方式

### 5.1 首次 + 开发

```bash
cd frontend
pnpm install                # 安装（会触发 esbuild 等构建脚本，由 pnpm-workspace.yaml allowBuilds 控制）
pnpm dev
```

- 脚本定义：`package.json` `"dev": "vite"`
- 默认监听 `http://localhost:5173`（若端口占用会自动 +1）
- **Vite dev proxy**（`vite.config.ts:14`）自动把 `/ws`、`/chat`、`/health`、`/api/*`、`/generate_wiki` 转发到 `ws://127.0.0.1:8000` / `http://...`，**彻底避免 CORS**。

### 5.2 其他命令

```bash
pnpm build                  # 生产构建 → dist/
pnpm preview                # 本地预览 dist/（需先 build）
pnpm lint                   # vue-tsc --noEmit 类型检查（无 emit）
```

从根目录执行（无需 cd）：

```bash
pnpm --dir frontend dev
pnpm --dir frontend lint
```

### 5.3 前端独立运行（不依赖 proxy）

```bash
# 在 frontend 目录创建 .env（Vite 自动加载 VITE_ 前缀）
VITE_API_BASE=http://127.0.0.1:8000
VITE_WS_BASE=ws://127.0.0.1:8000

pnpm build
pnpm preview
# 或用任意静态服务器 serve dist/
```

此时前端可部署到任意 origin，后端需允许 CORS（当前代码未加，生产需在 main.py 加 CORSMiddleware）。

## 6. 同时运行前后端（典型开发流程）

推荐开两个终端（或 tmux / 脚本）：

**终端 1（后端）：**

```bash
uv run python -m asuka.main
```

**终端 2（前端）：**

```bash
pnpm --dir frontend dev
```

浏览器打开 `http://localhost:5173`（或 proxy 分配的端口）。

- Header 健康徽标会每 5s 轮询 `/health` 变绿并显示版本。
- Chat 视图使用 WS 流式 + Live2D 驱动 + TTS。
- 可切到 `/agent` 测试非流式 `/chat`。
- 可切到 `/wiki` 使用独立 wiki 表单。

## 8. 数据流简述（启动后）

```
用户浏览器 (5173)
  │  (Vite proxy 仅 dev)
  ▼
Vite dev server
  │ 代理 /ws → ws://127.0.0.1:8000/ws/xxx
  │ 代理 /health 等 → http://127.0.0.1:8000/...
  ▼
FastAPI (uvicorn, 8000)
  │ lifespan → get_checkpointer() (mkdir data/ + AsyncSqliteSaver aenter)
  │ routes/ws.py → build_agent() → dispatch + LLM
  │ routes/ 其他
  └── checkpointer 自动 save/load 每轮 state（thread_id=conversation_id）
```

## 9. 常见问题与排查

- **port 8000 / 5173 被占用**：改 .env PORT，或杀进程；vite 会自动找空闲端口。
- **LLM 调用失败**：检查 .env Key 是否正确、余额、网络（DeepSeek 需可访问）；健康检查仍通过。
- **前端连不上后端**：确认后端已起；proxy 日志；或手动设 VITE_* 后重启 vite。
- **第一次 pnpm install 慢 / 失败**：检查网络（国内可用淘宝源），确认 allowBuilds 中的 esbuild 能编译。
- **DB 锁定**：多实例同时写同一 sessions.db 会冲突；开发期单实例。
- **类型 / lint 报错**：后端 `uv run mypy asuka/` + `uv run ruff check asuka/`；前端 `pnpm --dir frontend lint`。
- **无测试覆盖启动**：按 AGENTS 要求，验收时手动 curl + 浏览器 + `uv run pytest` 全绿。
