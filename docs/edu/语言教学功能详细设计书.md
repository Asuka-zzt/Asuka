# 语言教学功能详细设计书

> 依据：`docs/edu/语言教学设计.md`
>
> 本文描述 MVP 的工程落地方案。实现阶段应以本文为接口契约，除非发现现有代码存在不可兼容约束，再以追加说明方式更新。

---

## 1. 总体架构

语言教学功能接入现有单 Agent 对话链路，不新增独立教学服务。

```text
LanguageLearningView.vue
  选择 persona_id + level
  发送 WS 消息
    -> routes/ws.py
    -> resolve_agent_config(persona_id, level)
    -> dispatch.build_agent(config)
    -> create_agent + language tools
    -> token / tool.result / done
  渲染聊天、纠错卡、练习卡
  点击朗读
    -> POST /api/tts { text, language }
    -> edge-tts audio/mpeg
```

核心原则：
- persona 决定 Soul 和可用工具。
- level 只作为本轮上下文参数，不落库。
- 结构化纠错/出题通过 tool result 下发，普通陪练仍走流式 token。
- TTS 不交给 LLM 决策，前端直接调用 REST。

## 2. 后端设计

### 2.1 AgentConfig 扩展

文件：`asuka/core/agent/model.py`

新增可选字段，保持向后兼容：

```python
class AgentConfig(BaseModel):
    id: str = "default"
    name: str = "Asuka"
    soul: str = ...
    model_id: str = Field(default_factory=lambda: get_settings().default_model)

    language: str | None = None
    level: str | None = None
```

字段含义：
- `language`: `english` / `japanese`，默认 agent 为 `None`。
- `level`: 英语 CEFR 或日语 JLPT，MVP 由请求传入。

### 2.2 Persona 预置

文件：`asuka/core/agent/presets.py`

接口：

```python
SUPPORTED_PERSONA_IDS = {"default", "english_teacher", "japanese_teacher"}

def english_teacher(level: str = "B1") -> AgentConfig: ...
def japanese_teacher(level: str = "N4") -> AgentConfig: ...
def resolve_agent_config(persona_id: str | None, level: str | None) -> AgentConfig: ...
```

`resolve_agent_config` 规则：
- `None` 或 `default` -> `default_agent()`。
- `english_teacher` -> `english_teacher(level or "B1")`。
- `japanese_teacher` -> `japanese_teacher(level or "N4")`。
- 未知 persona id 抛出 `ValueError`，路由层转成 4xx 或 WS error。

英语 Soul 关键约束：
- 面向中文母语者。
- 中文解释，英语例句和改写。
- 优先纠错：冠词、时态、单复数/可数性、介词、主谓一致。
- 输出地道表达和 IPA。
- 用户要求批改时优先调用 `correct_text`。
- 用户要求练习时优先调用 `generate_quiz`。

日语 Soul 关键约束：
- 面向中文母语者。
- 中文解释，日语例句和改写。
- 优先纠错：助词、敬体/简体、敬语、自他动词、授受动词、长短音/促音。
- 输出假名和罗马音。
- 用户要求批改时优先调用 `correct_text`。
- 用户要求练习时优先调用 `generate_quiz`。

### 2.3 语言工具 schema

文件：`asuka/core/tools/schemas.py`

```python
from typing import Literal

from pydantic import BaseModel, Field


LanguageCode = Literal["english", "japanese"]
QuizType = Literal["mcq", "cloze", "translation"]


class CorrectionItem(BaseModel):
    error_type: str = Field(description="错误类型，如 冠词 / 时态 / 助词")
    original: str = Field(description="原始错误片段")
    corrected: str = Field(description="修正后的片段")
    explanation_zh: str = Field(description="中文解释")


class CorrectionResult(BaseModel):
    has_error: bool
    language: LanguageCode
    items: list[CorrectionItem]
    natural_rewrite: str
    annotation: str
    encouragement_zh: str


class QuizItem(BaseModel):
    type: QuizType
    question: str
    options: list[str] | None = None
    answer: str
    explanation_zh: str
    annotation: str


class QuizSet(BaseModel):
    language: LanguageCode
    level: str
    topic: str
    items: list[QuizItem]
```

约束：
- 工具返回值必须能通过 Pydantic 序列化。
- `items` 可以为空，但 `has_error=False` 时必须提供 `natural_rewrite`。
- `generate_quiz` 的 `items` 数量由 `n` 控制，路由/工具层将 `n` 限制在 1 到 10。

### 2.4 correct_text 工具

文件：`asuka/core/tools/language.py`

签名：

```python
@tool
async def correct_text(text: str, language: LanguageCode, level: str | None = None) -> CorrectionResult:
    """批改目标语文本，返回逐条纠错、地道改写、中文讲解和发音标注。"""
```

流程：
1. 校验 `text.strip()` 非空。
2. 校验 `language` 为 `english` 或 `japanese`。
3. 根据语言选择 checklist。
4. 构造批改 prompt。
5. 使用 `get_llm(get_settings().default_model).with_structured_output(CorrectionResult)`。
6. 返回 `CorrectionResult`。

Prompt 必须包含：
- 用户是中文母语者。
- 解释必须中文。
- 改写必须目标语。
- 英语给 IPA，日语给假名 + 罗马音。
- 不要扩写成课程，只做当前文本批改。

### 2.5 generate_quiz 工具

文件：`asuka/core/tools/language.py`

签名：

```python
@tool
async def generate_quiz(
    language: LanguageCode,
    level: str,
    topic: str = "日常",
    quiz_type: QuizType = "mcq",
    n: int = 5,
) -> QuizSet:
    """按语言、水平、主题和题型生成练习题。"""
```

校验：
- `language` 只能是 `english` / `japanese`。
- `quiz_type` 只能是 `mcq` / `cloze` / `translation`。
- `n` 限制在 1 到 10。
- 英语 level 建议 A1-C2；日语 level 建议 N5-N1。MVP 允许字符串，但 prompt 中要求模型按对应体系理解。

题型规则：
- `mcq`: `options` 必须至少 4 项。
- `cloze`: `question` 必须包含 `___`。
- `translation`: `options` 为 `None`。
- 每题必须有中文解析和发音/假名标注。

### 2.6 工具注册

文件：`asuka/core/tools/registry.py`

设计：

```python
_TEACHER_TOOLS: list[BaseTool] = [correct_text, generate_quiz]
_DEFAULT_TOOLS: list[BaseTool] = [generate_wiki]


def get_tools_for_agent(agent_id: str) -> list[BaseTool]:
    if agent_id in {"english_teacher", "japanese_teacher"}:
        return list(_TEACHER_TOOLS)
    return list(_DEFAULT_TOOLS)
```

验收：
- 语言老师不获得 `generate_wiki`。
- default agent 不获得语言教学工具。

### 2.7 Agent 构建与缓存

文件：`asuka/core/graph/dispatch.py`

当前单缓存：

```python
_agent: Any | None = None
```

目标多缓存：

```python
_agents: dict[str, Any] = {}


def _cache_key(cfg: AgentConfig) -> str:
    return f"{cfg.id}:{cfg.level or ''}:{cfg.model_id}"


async def build_agent(config: AgentConfig | None = None) -> Any:
    cfg = config or default_agent()
    key = _cache_key(cfg)
    if key not in _agents:
        _agents[key] = create_agent(
            model=get_llm(cfg.model_id),
            tools=get_tools_for_agent(cfg.id),
            system_prompt=cfg.soul,
            checkpointer=await get_checkpointer(),
        )
    return _agents[key]
```

说明：
- cache key 包含 level，因为 Soul 中注入了水平。
- 同一 persona 同一 level 复用 compiled agent。
- 后续如果 model/provider 参数更多，再扩展 key。

### 2.8 WebSocket 协议

文件：`asuka/routes/ws.py`

客户端请求：

```json
{
  "message": "给我出 5 道 N5 点餐题",
  "persona_id": "japanese_teacher",
  "level": "N5"
}
```

字段：
- `message`: 必填，非空字符串。
- `persona_id`: 可选，默认 `default`。
- `level`: 可选，英语默认 `B1`，日语默认 `N4`。

服务端事件新增：

```json
{
  "type": "tool.result",
  "name": "generate_quiz",
  "payload": {
    "language": "japanese",
    "level": "N5",
    "topic": "点餐",
    "items": []
  }
}
```

事件处理：
- `on_chat_model_stream`: 保持现有 token + Live2D 标签提取。
- `on_tool_end`: 若工具为 `correct_text` 或 `generate_quiz`，序列化 tool output 并发送 `tool.result`。
- 其他工具事件不下发给语言学习页，或后续统一成 debug 事件。

异常：
- 未知 persona -> 发送 `{"type": "error", "content": "unknown persona_id: ..."}`。
- 工具序列化失败 -> 发送 error，当前轮结束。

### 2.9 POST chat 调试接口

文件：`asuka/routes/chat.py`

请求：

```python
class ChatRequest(BaseModel):
    conversation_id: str
    message: str
    persona_id: str | None = None
    level: str | None = None
```

行为：
- 与 WS 使用同一个 `resolve_agent_config`。
- 返回完整 reply。
- 工具结果可暂不单独返回；若需要调试，扩展 `ChatResponse.tool_results`。

### 2.10 TTS 设计

文件：
- `asuka/config.py`
- `asuka/routes/tts.py`
- `asuka/api/provider/tts.py`

配置：

```python
tts_voice: str = "zh-CN-XiaoxiaoNeural"
tts_voices: dict[str, str] = {
    "english": "en-US-AriaNeural",
    "japanese": "ja-JP-NanamiNeural",
}
tts_rate: str = "-10%"
tts_volume: str = "+0%"
tts_pitch: str = "+0Hz"
tts_max_chars: int = 1200
```

请求：

```python
class TTSRequest(BaseModel):
    text: str = Field(..., min_length=1)
    language: Literal["english", "japanese"] | None = None
    voice: str | None = None
```

选音规则：
1. `voice` 非空时优先使用 `voice`。
2. 否则 `language` 非空时使用 `settings.tts_voices[language]`。
3. 否则使用 `settings.tts_voice`。

响应：
- `/api/tts`: `audio/mpeg` bytes。
- `/api/tts/stream`: `audio/mpeg` streaming。

兼容性：
- 保留旧请求 `{ "text": "...", "voice": "..." }`。
- 新学习页推荐请求 `{ "text": "...", "language": "japanese" }`。

## 4. 数据流细节

### 4.1 普通陪练

```text
用户输入
  -> WS { message, persona_id, level }
  -> resolve_agent_config
  -> build_agent
  -> create_agent astream_events
  -> on_chat_model_stream
  -> token
  -> 前端追加到 assistant message
```

### 4.2 结构化纠错

```text
用户：“帮我批改 I has a apple.”
  -> english_teacher Soul 引导 agent 调 correct_text
  -> correct_text 返回 CorrectionResult
  -> ws.py 捕获 on_tool_end
  -> tool.result
  -> 前端渲染 CorrectionCard
  -> agent 继续生成自然语言总结 token
```

### 4.3 练习生成

```text
用户：“出 5 道 N5 点餐题”
  -> japanese_teacher 调 generate_quiz
  -> QuizSet
  -> tool.result
  -> 前端渲染 QuizCard
  -> 题干旁 TTS 按钮可朗读
```

### 4.4 TTS

```text
用户点击朗读
  -> POST /api/tts { text, language }
  -> _resolve_voice(language, voice)
  -> edge_tts.Communicate
  -> audio/mpeg
  -> 前端播放
```

## 5. 错误处理

| 场景 | 后端行为 | 前端行为 |
|---|---|---|
| 空消息 | WS 忽略或 error | 不发送空消息 |
| 未知 persona | WS error / POST 422 | 显示错误 |
| 非法 language | 工具抛校验错误 | 显示工具失败 |
| 出题 n 过大 | clamp 到 10 或 422 | 限制输入 |
| TTS 文本超长 | 413 | 按钮显示失败 |
| TTS 服务失败 | 502 | 不影响聊天 |
| 工具结果无法序列化 | WS error | 显示工具失败 |

## 6. 安全与配置

- 不新增 API Key。
- LLM API Key 仍只从 `config.py` 读取环境变量。
- TTS 使用现有 `edge-tts`，不保存用户文本。
- 不允许工具读写用户文件。
- 语言工具只生成文本结构，不执行 shell、文件或网络访问。

## 10. 追加详细设计：WebRTC 实时语音链路（2026-06-03）

### 10.1 设计结论

WebRTC 可用于优化语言教学的实时语音链路，但只应覆盖「双向语音陪练」场景。

保留现有链路：
- 句子朗读按钮：继续使用 `/api/tts`。
- 文本聊天：继续使用 `/ws/{conversation_id}`。

新增链路：
- 实时语音陪练：使用 WebRTC 承载浏览器麦克风上行音频和老师语音下行音频，DataChannel 承载转写、老师文本、错误状态等结构化事件。

### 10.2 目标架构

```text
frontend VoiceBar
```

### 10.6 DataChannel 协议

前端和后端通过 DataChannel 传递 JSON。

```ts
export type VoiceDataChannelEvent =
  | { type: 'voice.ready' }
  | { type: 'transcript.partial', text: string }
  | { type: 'transcript.final', text: string }
  | { type: 'agent.text', text: string }
  | { type: 'agent.tool_result', name: string, payload: unknown }
  | { type: 'voice.error', content: string }
```

说明：
- `transcript.partial` 用于实时显示用户正在说什么。
- `transcript.final` 作为一轮用户消息进入 agent。
- `agent.text` 同步显示老师文本回复。
- `agent.tool_result` 复用语言教学工具结果，前端仍渲染纠错卡/练习卡。
- 老师语音不走 DataChannel，走 WebRTC 下行 audio track。

### 10.7 语音 pipeline

文件：`asuka/core/voice/pipeline.py`

```text
audio frames
  -> VAD / turn detection
  -> STT partial
  -> STT final
  -> HumanMessage(final_text)
  -> teacher agent astream_events
  -> agent.text / agent.tool_result data channel events
  -> TTS chunks
  -> downstream audio track
```

实现策略：
- 第一版可用手动按住说话或前端停止按钮触发 final，降低 VAD 难度。
- STT provider 先定义抽象并在测试中 mock，不在设计阶段绑定供应商。
- Agent 调用复用 `resolve_agent_config` 和 `build_agent`。
- 工具结果序列化逻辑应与 `routes/ws.py` 共用 helper，避免 WS 和 WebRTC 两套解析。

### 10.8 STT 抽象

文件：`asuka/api/provider/stt.py`

```python
class STTResult(BaseModel):
    text: str
    is_final: bool


class STTProvider(Protocol):
    async def accept_audio(self, pcm: bytes) -> STTResult | None: ...
    async def flush(self) -> STTResult | None: ...
```

MVP 之后的 WebRTC 阶段先接 mock provider：
- 单元测试直接输入固定音频 token，返回固定 transcript。
- 真正 STT 接入作为单独实现任务。

### 10.12 不做项

WebRTC 阶段仍不做：
- 口音评分细粒度打分。
- 音素级跟读对齐。
- 多人会议式语音房。
- 语音内容持久化存储。
- WebRTC 替换所有文本 WebSocket。
