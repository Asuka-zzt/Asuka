# 语言教学功能开发计划书

> 依据：`docs/edu/语言教学设计.md`
>
> 目标：在不重造 Agent、工具、TTS、前端聊天基建的前提下，交付「英语老师」「日语老师」两个语言教学 persona，支持对话陪练、即时纠错、结构化练习生成与目标语 TTS 朗读。

---

## 1. 项目目标

### 1.1 业务目标

- 面向中文母语者提供英语、日语两个独立语言老师 persona。
- 用户可以在同一个学习页中选择老师与水平，进行自由对话陪练。
- 老师在对话中即时指出错误、给出中文解释和目标语地道表达。
- 用户可以请求老师按水平生成练习或测验。
- 前端可在目标语句子旁触发 TTS 朗读，辅助听说练习。

### 1.2 技术目标

- 复用现有 LangGraph / LangChain `create_agent`、`@tool`、WebSocket 流式聊天、FastAPI 路由与 Vue 前端结构。
- 不引入 db/memory 大重构；MVP 只使用请求级水平参数和当前会话 checkpointer。
- 工具按 persona 分配，语言老师不拿 wiki 工具，默认助手继续保留 wiki 工具。
- 使用 Pydantic / TypeScript 类型定义承载结构化纠错与练习结果，避免裸 `dict` 在业务层扩散。

## 2. 范围

### 2.1 MVP 范围

| 模块 | 范围 |
|---|---|
| Persona | 新增 `english_teacher`、`japanese_teacher` 两个预置 AgentConfig |
| Prompt | 为两个老师编写 Soul，包含中文讲解、目标语沉浸、纠错 checklist、标注规则 |
| 工具 | 新增 `correct_text`、`generate_quiz` 两个语言教学工具 |
| 工具分配 | `get_tools_for_agent(agent_id)` 按 persona id 返回工具 |
| Agent 缓存 | `dispatch.build_agent()` 从单实例缓存升级为按 agent id 缓存 |
| WS 协议 | 客户端消息支持 `persona_id`、`level`；服务端支持结构化工具结果事件 |
| TTS | 复用现有 `/api/tts`，扩展 language 到 voice 映射，英语/日语默认音色可配置 |
| 前端 | 新增学习页，支持老师/水平选择、聊天、纠错卡、练习卡、朗读按钮 |
| 测试 | 后端工具、persona、registry、dispatch、TTS 路由、WS 事件；前端类型与关键交互 |

### 2.2 非 MVP 范围

- 生词卡与 SRS 间隔复习。
- 跨会话错误记忆与长期 RAG 注入。
- LearnerProfile 持久化。
- STT 跟读评分。
- 自适应教学 LangGraph 子图。
- 完整语法课程体系。
- TTS 缓存层或离线 TTS 引擎。

## 3. 当前基线

### 3.1 已具备能力

- `asuka/core/agent/model.py` 已有 `AgentConfig` 与 `default_agent()`。
- `asuka/core/graph/dispatch.py` 已使用 LangChain `create_agent` 构建可对话 Agent。
- `asuka/core/tools/registry.py` 已有工具注册入口，目前所有 agent 返回 wiki 工具。
- `asuka/routes/ws.py` 已支持 WebSocket 流式回复，并提取 Live2D 标签。
- `asuka/routes/tts.py` 与 `asuka/api/provider/tts.py` 已有 `edge-tts` 音频合成能力。
- `frontend/src/views/ChatView.vue`、`stores/chat.ts`、`useChatSocket.ts` 已有聊天基础设施。

### 3.2 需要升级的限制

- `dispatch.py` 目前只缓存单个 `_agent`，无法同时支持多个 persona。
- `AgentConfig` 缺少 `language`、`level` 等教学上下文字段。
- WS 客户端消息当前只有 `message`，无法选择 persona 和学习水平。
- WS 服务端目前只下发 token、done、error、Live2D 事件，尚未对工具结构化结果建事件。
- TTS 当前以 `voice` 为入参，默认音色是中文；语言教学需要 `language` 到默认音色的映射。

## 4. 里程碑计划

### Phase 0 - 设计确认

**目标：** 完成计划书与详细设计书，明确接口、文件改动、测试口径。

交付物：
- `docs/edu/语言教学功能开发计划书.md`
- `docs/edu/语言教学功能详细设计书.md`

完成标准：
- MVP 范围、非目标、数据结构、WS/TTS 接口、测试策略明确。
- 用户确认后再进入实现阶段。

### Phase 1 - 后端 persona 与工具

**目标：** 后端能按 persona 构建语言老师，并调用结构化语言工具。

任务：
- 在 `AgentConfig` 增加可选 `language`、`level` 字段。
- 新增 `core/agent/presets.py`，提供 `english_teacher(level)`、`japanese_teacher(level)`。
- 新增 `core/tools/schemas.py`，定义 `CorrectionResult`、`QuizSet` 等 Pydantic schema。
- 新增 `core/tools/language.py`，实现 `correct_text`、`generate_quiz`。
- 修改 `core/tools/registry.py`，语言老师只获得语言工具，默认 agent 继续获得 wiki 工具。

完成标准：
- 单元测试可断言 persona 字段、Soul 关键约束和工具分配。
- 语言工具使用 `with_structured_output`，测试中使用 fake/mock LLM，不发真实 LLM 请求。

### Phase 2 - Agent 调度与协议升级

**目标：** WS / POST chat 能根据请求中的 persona 和水平调用对应老师。

任务：
- 修改 `dispatch.py`，将 `_agent` 单缓存改为 `_agents: dict[str, Any]`。
- 新增 persona resolver，按 `persona_id + level` 生成 `AgentConfig`。
- 修改 `routes/ws.py` 客户端请求协议，接收 `persona_id`、`level`。
- 修改 `routes/ws.py` 事件处理，捕获 `on_tool_end` 并下发 `tool.result`。
- 修改 `routes/chat.py`，调试接口同步支持 persona 与 level。

完成标准：
- 同一进程内 `default`、`english_teacher`、`japanese_teacher` 可分别构建和缓存。
- WS 仍保持 token 流式输出，工具结果以独立事件下发。
- Live2D 标签提取行为不回退。

### Phase 3 - TTS 语言音色扩展

**目标：** 前端可以按英语/日语句子直接请求对应音色朗读。

任务：
- 扩展 `Settings`：`tts_voices: dict[str, str]`，保留现有 `tts_voice` 作为 fallback。
- 扩展 `TTSRequest`：新增 `language: Literal["english", "japanese"] | None`。
- Provider 根据 `language` 选择默认音色；显式 `voice` 仍可覆盖。
- 路由保留 `/api/tts` 与 `/api/tts/stream`，避免破坏现有前端 TTS。

完成标准：
- `/api/tts` 传 `language=english` 使用英语音色，传 `language=japanese` 使用日语音色。
- 空文本、超长文本、非法 language 有清晰 4xx 响应。
- 测试 mock provider，不访问外部 TTS 网络。

### Phase 4 - 前端学习页

**目标：** 用户可在一个完整页面完成老师选择、对话、纠错、练习和朗读。

任务：
- 新增 `frontend/src/types/language.ts`，对齐后端 Pydantic schema。
- 新增 `LanguageLearningView.vue`。
- 扩展 `useChatSocket` 或新增学习专用 composable，发送 `persona_id`、`level`。
- 扩展 `WsEvent`，处理 `tool.result`。
- 新增纠错卡片、练习卡片和目标语朗读按钮。
- 修改 router 与导航，增加学习入口。

完成标准：
- 英语老师 / 日语老师切换后，发送消息使用对应 persona。
- 纠错工具结果能渲染逐条错误、地道改写、中文解释和标注。
- 练习工具结果能渲染选择/填空/翻译题，并展示答案与中文解析。
- 目标语句子旁朗读按钮调用 `/api/tts`，传入正确 language。

### Phase 5 - 验收与收尾

**目标：** 后端、前端、类型、测试均达到可合并标准。

任务：

- 运行 `uv run pytest`。

- 运行 `uv run mypy asuka/`。
- 前端运行类型检查和构建。
- 补充二期 TODO 到 `docs/todo/未完成功能.md`。

完成标准：

- ruff / mypy 无 error。

## 6. 接口验收用例

### 6.1 英语陪练

输入：

```json
{
  "message": "I has a apple.",
  "persona_id": "english_teacher",
  "level": "A2"
}
```

期望：
- 老师回复中文指出主谓一致和冠词问题。
- 给出地道改写：`I have an apple.`
- 标注包含英语发音信息。

### 6.2 日语陪练

输入：

```json
{
  "message": "私は寿司を好きです。",
  "persona_id": "japanese_teacher",
  "level": "N5"
}
```

期望：
- 老师指出助词/表达问题。
- 给出自然表达：`私は寿司が好きです。`
- 标注包含假名和罗马音。

### 6.3 练习生成

输入：

```json
{
  "message": "给我出 5 道 N5 点餐选择题",
  "persona_id": "japanese_teacher",
  "level": "N5"
}
```

期望：
- 后端下发 `tool.result`，工具名为 `generate_quiz`。
- 前端显示 5 道题，包含答案、中文解析和标注。

### 6.4 TTS

输入：

```json
{
  "text": "私は寿司が好きです。",
  "language": "japanese"
}
```

期望：
- 返回 `audio/mpeg`。
- 使用日语默认音色，且不影响旧的 `voice` 覆盖参数。

## 8. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| LangChain 工具事件结构和版本差异 | WS `tool.result` 解析失败 | 测试固定事件样例；只读取稳定字段，保留 fallback |
| `with_structured_output` 在不同 provider 行为不一致 | 工具返回 schema 不稳定 | prompt 中强约束字段；测试 mock；运行时捕获并返回错误事件 |
| TTS 外部服务不稳定 | 朗读按钮失败 | 路由返回 502，前端显示播放失败，不阻塞聊天 |
| 日语标注质量不稳定 | 学习体验下降 | Soul 和工具 prompt 中明确假名/罗马音字段，前端独立展示 |
| 现有工作区有大量未提交改动 | 易误改用户变更 | 实现前切分支并只改 EDU 相关文件 |

## 11. 追加计划：WebRTC 语音链路优化（2026-06-03）

### 11.1 可行性判断

WebRTC 可行，但定位为「实时语音陪练」优化，不替代 MVP 的 REST TTS 朗读按钮。

原因：
- 朗读按钮是短文本按需播放，REST TTS 简单、稳定、可缓存。
- WebRTC 的优势是双向实时音频、低延迟播放、抖动控制和浏览器原生麦克风链路。
- 只做 TTS 播放时引入 WebRTC 成本高于收益；做麦克风实时陪练时收益明确。

### 11.2 计划调整

MVP Phase 1-5 不变。新增 Phase 6 作为确认后可启动的独立阶段。

### Phase 6 - WebRTC 实时语音陪练（可选二期）

**目标：** 在语言学习页增加实时语音模式，用户通过麦克风与英语/日语老师进行低延迟语音陪练。

任务：
- 新增 `routes/voice.py`，提供 WebRTC offer/answer 信令接口。
- 新增 `core/voice/session.py`，绑定 `conversation_id`、`persona_id`、`level` 和 PeerConnection 生命周期。
- 新增 `core/voice/pipeline.py`，串联 audio input、STT、Agent、TTS output。
- 新增 `api/provider/stt.py` 抽象，测试阶段先 mock，真实接入另行评估。
- 复用 `api/provider/tts.py`，支持把 TTS chunk 推入 WebRTC downstream audio track。
- 前端新增 `useVoiceSession.ts`，管理 `getUserMedia`、`RTCPeerConnection`、DataChannel。
- 学习页新增 `VoiceBar.vue`，提供开始/停止语音、静音、连接状态、转写显示。

完成标准：
- 本地浏览器授权麦克风后可建立 PeerConnection。
- 用户语音被转写为目标文本，并进入对应 teacher persona 的会话。
- 老师文本回复仍能下发到聊天区。
- 老师语音通过 WebRTC 播放。
- 连接失败、STT 失败、TTS 失败均有可见错误状态。

### 11.3 新增验收用例

英语语音陪练：
- 选择「英语老师 + A2」。
- 开启语音模式。
- 说出 `I has a apple`。
- 前端显示最终转写。
- 老师指出错误并给出 `I have an apple.`。
- 老师语音低延迟播放。

日语语音陪练：
- 选择「日语老师 + N5」。
- 开启语音模式。
- 说出一句点餐日语。
- 老师以中文解释 + 日语改写反馈。
- DataChannel 下发 `transcript.final` 和 `agent.text` 事件。

### 11.4 新增风险

| 风险 | 影响 | 应对 |
|---|---|---|
| WebRTC 需要 STUN/TURN | 公网环境连接失败 | 本地先 host candidate；部署阶段补 TURN 配置 |
| STT 未落地 | 无法完成语音输入闭环 | `api/provider/stt.py` 先抽象 + mock，真实 provider 独立评估 |
| edge-tts chunk 延迟不可控 | 老师语音不够实时 | MVP 保留 REST TTS；WebRTC 阶段实测，不达标则替换实时 TTS |
| 浏览器权限和设备差异 | 用户无法开麦 | 前端提供明确状态和降级到文本模式 |

### 11.5 依赖策略

- MVP 不新增 WebRTC 依赖。
- Phase 6 启动后再通过 `uv add aiortc` 增加后端 WebRTC 支持。
- STT provider 不在 Phase 6 设计中预设具体供应商，避免提前绑定外部服务。
